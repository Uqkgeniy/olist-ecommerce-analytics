# 🛒 Аналитика E-Commerce маркетплейса (Olist Dataset)

## 📌 О проекте
Проект посвящен анализу реальных данных бразильского маркетплейса Olist (100 000+ заказов). Основная цель — отработка сложных аналитических SQL-запросов и визуализация продуктовых метрик. Анализ проводился в среде Jupyter Notebook с использованием локальной базы данных SQLite.

**Исходные данные:** [Olist Brazilian E-Commerce Dataset (Kaggle)](https://www.kaggle.com/datasets/olistbr/brazilian-ecommerce)

## 🛠 Стек технологий
* **Языки:** SQL (диалект SQLite), Python
* **Библиотеки:** `pandas`, `sqlalchemy`, `matplotlib`, `seaborn`
* **SQL-навыки:** Window Functions (LAG, DENSE_RANK), CTE, Многотабличные JOIN, Агрегации, Работа с датами (strftime, julianday), CASE WHEN.

## 📊 Структура анализа
Проект включает 11 аналитических шагов:
1. Изучение ER-диаграммы и схемы БД.
2. Анализ географии клиентов (Базовый JOIN).
3. Динамика выручки по месяцам.
4. Воронка оттока по статусам заказов.
5. Расчет 7-дневного скользящего среднего выручки.
6. Выделение Топ-10 категорий товаров.
7. **Когортный анализ (Retention Rate)**.
8. Оценка темпов роста MoM (Month-over-Month).
9. Поиск Топ-3 селлеров в каждой категории.
10. Сводная профилирующая таблица по категориям.
11. Оценка влияния скорости доставки на NPS (оценки пользователей).

## 💡 Ключевые бизнес-инсайты
* **Retention:** У маркетплейса критически низкое удержание клиентов. Большинство пользователей совершают только одну покупку. Необходима стратегия реактивации и внедрение программ лояльности.
* **Логистика:** Выявлена прямая корреляция между скоростью доставки и удовлетворенностью. При доставке более 15 дней рейтинг падает ниже 3.7 баллов из 5.
* **География:** Основное ядро платежеспособной аудитории сконцентрировано в штате Сан-Паулу (SP), куда следует направлять основной маркетинговый бюджет.
