# 🛒 Аналитика E-Commerce маркетплейса (Olist Dataset)

## 📌 О проекте
Проект посвящен комплексному анализу реальных данных бразильского маркетплейса Olist (100 000+ заказов). Помимо SQL-аналитики и визуализации продуктовых метрик, проект включает моделирование бизнес-процессов в нотациях BPMN и UML на основе выявленных данных.

Анализ проводился в среде Jupyter Notebook с использованием локальной базы данных SQLite.

**Исходные данные:** [Olist Brazilian E-Commerce Dataset (Kaggle)](https://www.kaggle.com/datasets/olistbr/brazilian-ecommerce)

---

## 🛠 Стек технологий
* **Языки:** SQL (диалект SQLite), Python
* **Библиотеки:** `pandas`, `sqlalchemy`, `matplotlib`, `seaborn`
* **SQL-навыки:** Window Functions (LAG, DENSE_RANK), CTE, многотабличные JOIN, агрегации, работа с датами (strftime, julianday), CASE WHEN
* **Моделирование:** BPMN (нотация бизнес-процессов), UML Use Case

---

## 📊 Структура анализа
Проект включает 11 аналитических шагов:
1. Изучение ER-диаграммы и схемы БД
2. Анализ географии клиентов (базовый JOIN)
3. Динамика выручки по месяцам
4. Воронка оттока по статусам заказов
5. Расчёт 7-дневного скользящего среднего выручки
6. Выделение топ-10 категорий товаров
7. **Когортный анализ (Retention Rate)**
8. Оценка темпов роста MoM (Month-over-Month)
9. Поиск топ-3 продавцов в каждой категории
10. Сводная профилирующая таблица по категориям
11. Оценка влияния скорости доставки на NPS (оценки пользователей)

## 🗂 Моделирование бизнес-процессов

По результатам анализа данных смоделированы два артефакта:

### BPMN — процесс обработки заказа
Диаграмма описывает сквозной процесс от оформления заказа до получения отзыва. Участники: Покупатель, Olist, Продавец, Логистика.

Ключевые точки процесса привязаны к полям БД:
- `order_purchase_timestamp` — момент оформления заказа
- `order_approved_at` — подтверждение оплаты
- `shipping_limit_date` — дедлайн отгрузки для продавца
- `order_delivered_carrier_date` — передача в логистику
- `order_delivered_customer_date` — доставка покупателю
- `review_creation_date` — создание отзыва

Смоделирован сценарий неуспешной оплаты: покупатель получает уведомление и имеет 30 минут на повторную попытку, после чего заказ отменяется.

### UML Use Case — акторы и функции системы
Диаграмма описывает, кто и что делает в системе Olist. Акторы: Покупатель, Продавец, Система оплаты, Менеджер Olist. Связи include/extend отражают зависимости между сценариями использования.

## 💡 Ключевые бизнес-инсайты
* **Retention:** У маркетплейса критически низкое удержание клиентов. Большинство пользователей совершают только одну покупку. Необходима стратегия реактивации и внедрение программ лояльности.
* **Логистика:** Выявлена прямая корреляция между скоростью доставки и удовлетворённостью. При доставке более 15 дней рейтинг падает ниже 3.7 баллов из 5.
* **Geography:** Основное ядро платёжеспособной аудитории сконцентрировано в штате Сан-Паулу (SP) — сюда следует направлять основной маркетинговый бюджет.
* **BPMN-инсайт:** Основная задержка процесса возникает между подтверждением заказа и передачей в логистику, что коррелирует с оттоком в когортном анализе.
